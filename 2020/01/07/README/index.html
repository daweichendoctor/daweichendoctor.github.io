React Hooks

概念

  1. 16.8正式发布
  2. react hook是一种特殊的函数 其本质可以是函数式组件（返回dom或dom及state）,也可以只是一个工具函数（传入配置项返回封装后的数据处理逻辑）。
  3. 基本钩子 useSatate useEffect useContext
    扩展钩子 useCallback useReducer useMemo useRef useLayoutEffect useImperativeHandle useDebugValue

    使用

    规则

  4. 只在最顶层使用hook 不在循环、条件或嵌套中调用hook
    react怎么知道哪个state对应哪个useState? 答案是react靠的是Hook调用的顺序
  5. 只在react函数中调用Hook 不在普通的js函数中调用hook
  6. 自定义hook以use开头 驼峰

不遵循的话 由于无法判断某个函数是否包含对其内部Hook的调用 react将无法自动检查你的hook是否违反了hook的规则

FAQ

  1. lint规则强制了啥
    1. 对Hook的调用要么在大驼峰命名的函数(视作一个组件)内部，要么在useSomething函数(视作一个自定义hook)中
    2. Hook在每次渲染时都按照相同的顺序被调用
  2. hook中有类似实例变量的东西吗
    使用useRef(), 可以认为refs就像是一个class的实例变量
  3. 怎么只在更新时运行effect —使用一个可变的ref手动存储一个布尔值来表示是首次渲染还是后续渲染
  4. 如何获取上一轮的props或state – useEffect + useRef(见index)
  5. 为什么异步函数中获取的state可能是陈旧的(不是最新的)
    1. 组件内的任何函数包括事件处理和effect 都是从他被创建的那次渲染中获state
    2. 如何取最新的state？？
    3. 每次更新state 都将state都存到ref中 在异步函数中获取ref
  6. 怎么实现forceUpdate
    1. 如果前后两次值相同 useState和useReducer hook都会放弃更新。 原地state并调用setState不会引起重新渲染
    2. 利用useReducer
      const [ignored, forceUpdate] = useReduc(ignored => ignored + 1, 0);
